🔄 액션 정의

🔹 useEffect()

초기 렌더링 시 기사 상세 정보를 서버에서 가져옵니다. 현재 페이지 번호도 Redux를 통해 저장합니다.

① location.state에서 가져온 articleId 값이 존재하는지 체크합니다.

값이 존재하지 않는 경우, 에러 메세지 표시 후, 기사 일람으로 이동.

alert('記事情報がありません。');
navigate('/ArticleList');

② 서버에 기사 상세 정보를 요청합니다.

📄 記事詳細API 설계서


✅ 위의 결과에 에러가 발생하지 않은 경우
  1. 레스폰스로 받은 데이터를 상세 정보 state에 저장합니다.
// 記事情報
const [articleInfo, setArticleInfo] = useState({});

↓

// useEffect()
const response = await axios.get(`/api/article_servlet/detail/${articleId}`);
setArticleInfo(response.data);

  1. redux에서 관리하고 있는 현재 페이지 번호를 갱신합니다.
dispatch(setPaging({
    currentPage: nowPage,
}));

  1. 로컬의 loading state를 false로 갱신하여 화면이 리렌더링 될때 Loading... 문구가 표시되지 않도록합니다.
// state 선언
const [loading, setLoading] = useState(true)

↓

// useEffect()
setLoading(false);

↓

// 화면 표시
if (loading) {
    return <div>Loading...</div>; // 로딩 중 메시지
}

❌ 위의 결과에 에러가 발생한 경우 : 에러 메세지 출력

코드에러 메세지
400記事詳細情報を取得時に問題が発生しました。
500サーバー（DB）側でエラーが発生しました。

🔹 back()

기사 일람으로 이동합니다.

const back = () => {
    navigate('/ArticleList');
};

🔹 articleEdit()

기사 편집 화면으로 이동합니다. 이때 기사 아이디를 state로 전달합니다.
※admin 권한의 유저만 가능합니다.

const articleEdit = () => {
    navigate('/articleEdit', { state: { articleId: articleInfo.articleId } });
};

🔹 articleDelete()

기사 삭제 화면면으로 이동합니다. 이때 기사 아이디를 state로 전달합니다.
※admin 권한의 유저만 가능합니다.

const articleDelete = () => {
    navigate('/articleDelete', { state: { articleId: articleInfo.articleId } });
};
